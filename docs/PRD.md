# Jiphyeon — PRD

## 개요

Jiphyeon 마크다운 볼트를 위한 읽기 전용 데스크톱 대시보드. Tauri v2 기반으로 볼트 통계 시각화, 노트 탐색, 링크 그래프, 웹 클리핑 기능을 제공한다.
편집은 외부 에디터(VS Code 등)에서 수행하며, 대시보드는 파일 변경을 실시간 감지하여 자동 갱신한다.

## 문제 정의

- 마크다운 볼트는 텍스트 파일 모음이라 전체 구조를 한눈에 파악하기 어려움
- 노트 간 링크 관계(백링크)를 확인하려면 수동으로 grep 해야 함
- 태그/상태별 분포, 갭 분석 등 메타 정보를 시각적으로 볼 수 없음
- 웹 아티클 클리핑 시 별도 도구가 필요하고, 볼트 컨벤션(frontmatter, 파일명)에 맞추기 번거로움
- `_maintenance/reports/` 감사 리포트를 열어보기 불편함

## 대상 사용자

- Jiphyeon를 사용하는 개인 지식 관리자
- 터미널 + 에디터 중심 워크플로우에서 시각적 보완이 필요한 경우

## 핵심 기능

### 1. 대시보드 (Dashboard)

볼트 전체 현황을 한 화면에 표시한다.

- **노트 통계**: 전체 노트 수, 타입별 분포 (bar/pie chart), 상태별 분포 (seedling/growing/evergreen/stale)
- **태그 히트맵**: 태그 사용 빈도를 treemap 또는 히트맵으로 시각화. 계층 태그(`#domain/backend`) 지원
- **최근 노트**: 최근 생성/수정된 노트 목록 (최대 20개). 클릭 시 Markdown Viewer로 이동
- **링크 그래프**: 볼트 전체 노트의 wikilink 관계를 force-directed graph로 시각화. 노드 클릭 시 해당 노트로 이동. 노드 색상은 타입별 구분, 크기는 링크 수 비례
- **감사 결과**: 최신 감사 요약 표시 (고아 노트, 깨진 링크, stale 노트 수)
- **갭 분석**: 링크는 있으나 파일이 없는 wikilink 목록, 태그 카테고리별 불균형

### 2. 마크다운 뷰어 (Markdown Viewer)

노트를 읽기 전용으로 렌더링한다.

- YAML frontmatter를 파싱하여 메타데이터 영역에 구조화 표시 (타입 배지, 상태 배지, 태그 칩, 생성일)
- 마크다운 본문을 HTML로 렌더링 (pulldown-cmark, Rust 측에서 변환)
- `[[wikilink]]`를 클릭 가능한 내부 링크로 변환. 클릭 시 해당 노트로 네비게이션
- 코드 블록 구문 강조
- "외부 에디터에서 열기" 버튼

### 3. 백링크 패널 (Backlink Panel)

노트 선택 시 사이드 패널에 링크 관계를 표시한다.

- **Outgoing links**: 현재 노트가 참조하는 wikilink 목록
- **Incoming links (backlinks)**: 현재 노트를 참조하는 다른 노트 목록
- 각 항목은 노트 제목 + 타입 배지 + 링크가 포함된 주변 텍스트(컨텍스트). 클릭 시 해당 노트로 이동
- 깨진 링크(대상 파일 없음)는 경고 표시

### 4. 웹 클리핑 (Web Clipper)

URL을 입력하면 웹 페이지를 마크다운으로 변환하여 `inbox/`에 저장한다.

- URL 입력 필드 + "클립" 버튼
- 백엔드에서 `reqwest`로 HTML 가져오기 → `scraper`로 본문 추출 → `html2md`로 마크다운 변환
- `_templates/tpl-clipping.md` 기반 frontmatter 자동 생성:
  - `type: clipping`, `created`: 현재 날짜, `source`: 원본 URL
  - `author`: `<meta>` 태그에서 추출 (없으면 빈 문자열)
  - `status: seedling`, `tags: []`
- 파일명: `{YYYY-MM-DD}-{slugified-title}.md`
- 저장 경로: `inbox/`
- 저장 후 성공 토스트 + 바로 뷰어에서 열기 옵션

### 5. 실시간 파일 감시 (File Watcher)

볼트 파일 변경을 감지하여 UI를 자동 갱신한다.

- `notify` v6 크레이트로 볼트 루트 디렉토리 재귀 감시
- 감시 대상: `*.md` 파일의 생성, 수정, 삭제, 이름 변경
- 제외: `dashboard/`, `.git/`, `artifacts/`
- 변경 감지 시 해당 노트만 재파싱 (incremental update)
- 디바운스: 500ms (빠른 연속 저장 병합)
- 프론트엔드에 Tauri 이벤트로 변경 사항 전달 → 해당 뷰 자동 갱신

### 6. 노트 탐색 (Explorer)

파일 트리 또는 리스트 형태로 볼트를 탐색한다.

- 폴더별 트리 뷰 (inbox, dev, decisions, readings, meetings, ideas, projects, _moc)
- 리스트 뷰: 정렬 (생성일, 수정일, 타입, 상태), 필터 (타입, 상태, 태그)
- 검색: 파일명 + frontmatter 필드 기반 퀵 필터
- 노트 클릭 시 Markdown Viewer + Backlink Panel 표시

## 비기능 요구사항

### 성능

| 항목 | 목표 |
|------|------|
| 앱 시작 → 대시보드 표시 | < 2초 (1,000개 노트 기준) |
| 볼트 전체 파싱 | < 1초 (1,000개 노트) |
| 파일 변경 감지 → UI 반영 | < 1초 (디바운스 500ms 포함) |
| 마크다운 렌더링 | < 100ms (단일 노트) |
| 웹 클리핑 (네트워크 제외) | < 500ms |
| 링크 그래프 렌더링 | < 3초 (500 노드) |

### 메모리

- 유휴 시: < 100MB
- 볼트 인덱스 (1,000개 노트): < 10MB

### 안정성

- 파일 파싱 실패 시 해당 노트만 스킵, 전체 로딩 중단하지 않음
- frontmatter가 없는 마크다운도 제목/경로로 표시
- 잘못된 YAML frontmatter는 경고 배지 표시

## 제약 조건

- 읽기 전용: 노트 내용 편집 기능 없음 (웹 클리핑 저장만 예외)
- macOS 우선 (Tauri v2이므로 크로스플랫폼 가능하나, macOS에서만 테스트)
- 볼트 경로: 기본값은 dashboard의 부모 디렉토리 (`../`)
- 오프라인 동작: 웹 클리핑 외 모든 기능은 네트워크 불필요
