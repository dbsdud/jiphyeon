# v1.0 MVP 공개 배포 로드맵

> 목표: 정식 공개 배포 가능한 수준의 완성도
> 철학: 앱은 대시보드/뷰어, Claude Code(볼트 내)는 LLM 위키 엔진
> 범위: UI 전면 개선 + 대시보드 강화 + 녹취/영상 인제스트 + 볼트 스킬 정교화

## 배경

v0.5까지 백엔드 엔진(파서/인덱서/렌더러/검색)과 Claude 도구 뷰, 멀티볼트 관리가 완성되었다.
정식 공개 배포를 위해서는 다음 4개 축의 개선이 필요하다:

1. **UI 완성도** — 공개 배포의 첫인상. 노션 수준의 가벼움 + GitHub 수준의 마크다운 가독성
2. **대시보드 차별화** — Graphify식 그래프 인텔리전스(핵심 노드, 클러스터) 도입
3. **소스 범위 확장** — 녹취/영상을 볼트 노트로 인제스트 (Karpathy LLM 위키의 Ingest 축 강화)
4. **볼트 스킬/훅 품질** — 스캐폴드되는 14개 스킬 + 11개 훅의 실전 검증

## Epic 구조

```
Epic 1 ─ UI 전면 개선 (디자인 시스템 + GitHub MD + 사이드바)
Epic 2 ─ 대시보드 강화 (God Node + 클러스터 + 헬스 지표)
Epic 3 ─ 녹취/영상 인제스트 (Whisper + YouTube 자막)
Epic 4 ─ 볼트 스킬/훅 정교화 + 온보딩 개선
```

순서는 `Epic 1 → 2 → 3 → 4`. UI 기반이 선행되어야 나머지 Epic의 시각적 산출물이 일관성을 유지한다.

## Epic 1 — UI 전면 개선

**왜 먼저**: 정식 공개 배포의 첫인상. 나머지 Epic의 결과물이 모두 UI를 통해 드러난다.

**Slice**:
- 1.1 디자인 시스템 토큰 (색상/타이포그래피/간격/밀도)
- 1.2 컴팩트 모드 토글 (설정 persist + 전역 적용)
- 1.3 GitHub 스타일 마크다운 렌더러 (GFM, 코드 하이라이팅, 테이블)
- 1.4 사이드바 네비게이션 재구성 (노션 라이트 레이아웃)

**상세 플랜**: [`docs/plans/v1.0-epic1-ui.md`](v1.0-epic1-ui.md)

**완료 기준**:
- 디자인 토큰이 CSS 변수로 정의되고 Tailwind 4 `@theme`에서 참조됨
- 설정 → 컴팩트 모드 토글 → 모든 페이지 간격/글꼴 크기 축소 반영
- 마크다운 뷰어가 GitHub 스타일과 동등한 가독성
- 사이드바 구조가 명확한 위계를 가짐 (볼트/탐색/도구 그룹 분리)

## Epic 2 — 대시보드 강화

**왜**: Jiphyeon의 차별화 포인트. 마크다운 볼트 뷰어는 많지만 그래프 인텔리전스를 가진 앱은 드물다.

**Slice**:
- 2.1 God Node 카드 (링크 수 상위 노트 = 핵심 개념)
- 2.2 클러스터 지표 (연결 컴포넌트 기반 커뮤니티 탐지)
- 2.3 볼트 헬스 카드 (고립 노트, 깨진 링크, 최근 편집 활동)
- 2.4 최근 활동 타임라인 (파일 수정 이벤트 기반)

**선행 조건**: Epic 1 완료 (카드 디자인 시스템)

**완료 기준**:
- 대시보드 진입 시 God Node 상위 5개 노출
- 클러스터 수 + 각 클러스터 대표 노트 표시
- 고아 노트/깨진 링크를 클릭하면 해당 노트로 이동

## Epic 3 — 녹취/영상 인제스트

**왜**: Karpathy LLM 위키의 "소스 → 위키" 파이프라인에서 현재 지원되지 않는 영역. MVP 차별화.

**Slice**:
- 3.1 Whisper 로컬 실행 (tauri sidecar로 whisper.cpp 바이너리 번들)
- 3.2 로컬 오디오/비디오 파일 드래그 → 전사 → 노트 생성
- 3.3 YouTube/웹 URL → 자막 추출 → 노트 생성
- 3.4 `.claude/skills/vault-transcribe/` 스킬 추가 (Claude가 전사 결과를 요약/연결)

**선행 조건**: Epic 1 완료

**열린 결정**:
- Whisper 모델 크기 (tiny/base/small) — 번들 크기 vs 정확도
- YouTube 자막 추출 방식 — yt-dlp 번들 vs YouTube Transcript API
- 전사 결과 저장 경로 — `_sources/transcripts/` 하위 고정 vs 설정 가능

**완료 기준**:
- 5분짜리 mp3 파일 드롭 → 1분 내 전사 완료 → 볼트 노트로 저장
- YouTube URL 입력 → 자막 있으면 추출, 없으면 음성 다운로드 후 전사

## Epic 4 — 볼트 스킬/훅 정교화 + 온보딩

**왜**: 볼트 스캐폴드가 실제 LLM 위키 워크플로우를 지원해야 완결. MVP 릴리즈의 마지막 품질 검증.

**Slice**:
- 4.1 14개 스킬(SKILL.md) 각각 실제 볼트에서 동작 검증 및 프롬프트 개선
- 4.2 11개 훅 스크립트 에러 케이스 보강 (Claude Code 실제 실행 로그 기반)
- 4.3 온보딩 플로우 개선 (첫 볼트 생성 → 샘플 노트 → 스킬 데모)
- 4.4 `vault-transcribe` 스킬 정착 (Epic 3 산출물과 연동)

**선행 조건**: Epic 3 완료

**완료 기준**:
- 각 스킬이 최소 1개의 실제 사용 시나리오에서 검증됨
- 훅이 정상/에러 케이스 모두에서 볼트를 깨뜨리지 않음
- 첫 실행 → 3분 내 첫 노트 작성까지 도달

## 열린 결정

- **앱 이름 확정**: 현재 사이드바에 "Co-Vault", package.json/tauri.conf.json 확인 필요. v1.0 전에 "Jiphyeon"으로 통일
- **다크/라이트 테마**: 현재 다크 고정. 라이트 모드는 v1.0 이후로 보류 가능
- **배포 플랫폼**: macOS 우선, Windows/Linux는 v1.0 이후

## 위험

- **Whisper sidecar 크기**: tiny 모델도 ~75MB. 번들 vs 런타임 다운로드 결정 필요
- **렌더링 번들 크기**: GFM + 하이라이팅 + KaTeX + Mermaid + DBML 합쳐 ~1.3MB. Lazy-load로 완화
- **디자인 시스템 재작업 비용**: Epic 1이 길어지면 후속 Epic 일정 압박. Slice 단위로 ship 판단
