# v1.0 MVP 공개 배포 로드맵

> 목표: 정식 공개 배포 가능한 수준의 완성도
> 철학: 앱은 대시보드/뷰어, Claude Code(볼트 내)는 LLM 위키 엔진
> 범위: UI 전면 개선 + 대시보드 강화 + 녹취/영상 인제스트 + 볼트 스킬 정교화

## 배경

v0.5까지 백엔드 엔진(파서/인덱서/렌더러/검색)과 Claude 도구 뷰, 멀티볼트 관리가 완성되었다.
정식 공개 배포를 위해서는 다음 4개 축의 개선이 필요하다:

1. **UI 완성도** — 공개 배포의 첫인상. 노션 수준의 가벼움 + GitHub 수준의 마크다운 가독성
2. **대시보드 차별화** — Graphify식 그래프 인텔리전스(핵심 노드, 클러스터) 도입
3. **소스 범위 확장** — 녹취/영상을 볼트 노트로 인제스트 (Karpathy LLM 위키의 Ingest 축 강화)
4. **볼트 스킬/훅 품질** — 스캐폴드되는 14개 스킬 + 11개 훅의 실전 검증

## Epic 구조

```
Epic 1 ─ UI 전면 개선 (디자인 시스템 + GitHub MD + 사이드바)
Epic 2 ─ 대시보드 강화 (God Node + 클러스터 + 헬스 지표)
Epic 3 ─ 녹취/영상 인제스트 (Whisper + YouTube 자막)
Epic 4 ─ 볼트 스킬/훅 정교화 + 온보딩 개선
```

순서는 `Epic 1 → 2 → 3 → 4`. UI 기반이 선행되어야 나머지 Epic의 시각적 산출물이 일관성을 유지한다.

## Epic 1 — UI 전면 개선

**왜 먼저**: 정식 공개 배포의 첫인상. 나머지 Epic의 결과물이 모두 UI를 통해 드러난다.

**Slice**:
- 1.1 디자인 시스템 토큰 (색상/타이포그래피/간격/밀도)
- 1.2 컴팩트 모드 토글 (설정 persist + 전역 적용)
- 1.3 GitHub 스타일 마크다운 렌더러 (GFM, 코드 하이라이팅, 테이블)
- 1.4 사이드바 네비게이션 재구성 (노션 라이트 레이아웃)

**상세 플랜**: [`docs/plans/v1.0-epic1-ui.md`](v1.0-epic1-ui.md)

**완료 기준**:
- 디자인 토큰이 CSS 변수로 정의되고 Tailwind 4 `@theme`에서 참조됨
- 설정 → 컴팩트 모드 토글 → 모든 페이지 간격/글꼴 크기 축소 반영
- 마크다운 뷰어가 GitHub 스타일과 동등한 가독성
- 사이드바 구조가 명확한 위계를 가짐 (볼트/탐색/도구 그룹 분리)

**왜**: Jiphyeon의 차별화 포인트. 마크다운 볼트 뷰어는 많지만 그래프 인텔리전스를 가진 앱은 드물다.

**Slice**:

- 2.5 그래프 페이지 인텔리전스 (대시보드 지표의 시각적 확장)

**제외된 Slice**:

- ~~2.4 최근 활동 타임라인~~ — 대시보드 하단 Recent Notes 리스트로 충분. 히트맵은 우선순위 낮다고 판단해 드롭.

- 그래프 페이지에서 클러스터 색상, God Node 크기, 검색/필터/포커스 지원

### Slice 2.5 — 그래프 페이지 인텔리전스

**왜 별도 Slice**: 대시보드(Slice 2.1~2.4)는 **상태 요약**, 그래프 페이지는 **시각적 탐색**. 같은 집계 데이터를 두 층위로 보여주되 디자인/UX 결정이 겹치지 않도록 분리.

**선행 조건**: Slice 2.1(God Node), 2.2(클러스터) 완료 — 그래프 오버레이가 이들 집계 데이터를 재사용함.

**범위**:
- A. 클러스터 색상 분리 + 범례 (CC별 HSL 팔레트 순환, 범례에 상위 N개 표시)
- B. God Node 크기 스케일 (`backlink_count` 비례, log 또는 sqrt)
- C. 검색/필터 (노트 검색 → 노드 focus/pan, tag/type/folder 필터 dim)
- D. 포커스 모드 (노드 클릭 → 이웃 N-hop만 확대 표시)
- E. 품질 오버레이 (깨진 링크 붉은 점선, 고립 노트 페이드 — 기존 `get_orphan_notes`/`broken_links` 재활용)

**연동**:
- 대시보드 Slice 2.2 카드 → `/graph?cluster=<id>` 해당 클러스터 focus
- Slice 2.1 God Node 카드 → `/graph?focus=<path>` 포커스 모드 진입

**열린 결정**:
- 클러스터 색상 팔레트 (다크/라이트 가독성 동시 확보)
- N-hop 기본값 (1 vs 2)
- 검색 UI 위치 (그래프 상단 고정 바 vs 사이드 패널)
- 범례 토글 가능 여부

## Epic 3 — Transcribe: 녹음 + 볼트 스킬 연동

**철학 적용**: Jiphyeon = 녹음 캡처/저장, 볼트 Claude 스킬 = 전사/요약/노트화. 앱은 소스 파일을 볼트에 **떨어뜨리는 역할만** 담당하고, LLM 작업(Whisper 호출, 요약, md 생성)은 볼트의 `.claude/skills/vault-transcribe/` 스킬이 수행.

**Slice**:
- 3.1 사이드바 "🎙️ Transcribe" 활성화 + 기본 페이지 (현재 placeholder 상태 전환)
- 3.2 녹음 UI (MediaRecorder API, 파형 표시, record/stop)
- 3.3 녹음본 볼트 저장 (`_sources/recordings/YYYY-MM-DD-HHmm.webm` 등 경로 정책)
- 3.4 파일 투입 UX — Ctrl/Cmd+V 붙여넣기 (오디오 파일 클립보드에서)
- 3.5 `vault-transcribe` 스킬 scaffold 템플릿 — `rescaffold` 인프라에 추가 (실제 스킬 로직은 볼트 레포 작업)

**앱 측 범위에서 제외** (볼트 레포 작업):
- `openai-whisper` CLI 실행 — 스킬이 처리
- 전사본 요약/섹션 생성 — Claude가 스킬 프롬프트 따라 처리
- 최종 md 작성 + wikilink 연결 — 스킬이 처리
- YouTube 자막 추출 — 별도 볼트 스킬 (`vault-yt-transcribe` 등)로 후속 검토

**선행 조건**: Epic 1 완료

**열린 결정**:
- 녹음 포맷: `webm/opus`(브라우저 기본) vs `wav` 변환
- 파형 표시: Web Audio API AnalyserNode 기반 canvas vs SVG
- 전사 트리거 방식: (a) 수동 — 사용자가 Claude에게 요청, (b) 앱에서 볼트 스킬 직접 호출, (c) 파일 watcher가 `_sources/recordings/` 감지 후 훅 트리거
- 녹음 중 최대 시간 제한

**완료 기준** (앱 측):
- 사이드바 Transcribe 클릭 → 녹음 페이지 진입
- Record 버튼 → 파형 노출 → Stop → 볼트 `_sources/recordings/` 저장
- Ctrl/Cmd+V로 오디오 파일 붙여넣기 → 동일 경로 저장
- `rescaffold`에 `vault-transcribe` 스킬 템플릿 포함

## Epic 4 — Explore 개선 + 온보딩 + 릴리즈 준비

**왜**: 공개 배포 전 앱 체감 품질 정비. 탐색 UX의 잔 버그와 온보딩 첫인상이 MVP 완성도를 좌우.

**Slice**:
- 4.1 Explore 트리 개선
  - `.` 루트 노드 라벨 명확화 (`📓 (root)` 또는 `볼트 루트` 등)
  - `_moc` / `_` prefix 특수 폴더는 노출하되 **시각적 구분** (dim 톤, 들여쓰기 축소 또는 아이콘)
  - 접기/펼치기 chevron을 SVG 14px로 교체, `text-fg` 기본 + hover `text-accent`, 클릭 영역을 행 전체로 확장
  - 접힘 상태 `localStorage["explore-expanded"]` persist (theme/sidebar 패턴 재사용)
- 4.2 온보딩 플로우 개선 (첫 볼트 생성 → 샘플 노트 → 스킬/훅 데모)
- 4.3 README 정비 + 집현전 연혁에 **세종대왕 맥북 던짐 사건** 이스터에그 자연스럽게 삽입 (AI hallucination 밈 + 훈민정음 창제 배경)
- 4.4 v1.0 릴리즈 체크리스트 (license, CHANGELOG, 스크린샷, 설치 가이드)

**볼트 레포 작업으로 이관** (Jiphyeon 로드맵에서 제외):
- ~~14개 스킬 SKILL.md 프롬프트 개선~~ → 볼트 레포
- ~~11개 훅 스크립트 에러 케이스 보강~~ → 볼트 레포
- ~~`vault-transcribe` 스킬 본체~~ → 볼트 레포 (Jiphyeon은 scaffold 템플릿만 제공)

**선행 조건**: Epic 3 완료

**완료 기준**:
- 각 스킬이 최소 1개의 실제 사용 시나리오에서 검증됨
- 훅이 정상/에러 케이스 모두에서 볼트를 깨뜨리지 않음
- 첫 실행 → 3분 내 첫 노트 작성까지 도달

## 열린 결정

- **앱 이름 확정**: 현재 사이드바에 "Jiphyeon", package.json/tauri.conf.json 확인 필요. v1.0 전에 "Jiphyeon"으로 통일
- **다크/라이트 테마**: 현재 다크 고정. 라이트 모드는 v1.0 이후로 보류 가능
- **배포 플랫폼**: macOS 우선, Windows/Linux는 v1.0 이후

## 위험

- **Whisper sidecar 크기**: tiny 모델도 ~75MB. 번들 vs 런타임 다운로드 결정 필요
- **렌더링 번들 크기**: GFM + 하이라이팅 + KaTeX + Mermaid + DBML 합쳐 ~1.3MB. Lazy-load로 완화
- **디자인 시스템 재작업 비용**: Epic 1이 길어지면 후속 Epic 일정 압박. Slice 단위로 ship 판단
