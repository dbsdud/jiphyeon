## Slice A — 에디터 설정

### A1. `detect_editors()` 커맨드 (백엔드)

**시그니처 스케치** (상세 Spec은 spec-settings.md에서 확정)
- `pub fn detect_editors() -> Vec<DetectedEditor>`
- `DetectedEditor { id, label, kind, command }`
  - `id`: "vscode" | "cursor" | "zed" | "nvim" | "sublime" | "obsidian"
  - `kind`: `Command`(실행파일) | `Url`(Obsidian URL 스킴)
- 플랫폼 분기 (macOS 우선, Unix/Windows는 베스트 에포트)
  - macOS: `/Applications/*.app` 존재 확인 + `which`
  - Unix: `which code`, `which cursor`, `which zed`, `which nvim`, `which subl`
- 설치 안 된 에디터는 목록에서 제외

**TDD 테스트 대상**
- 반환 타입 직렬화 왕복
- Presence 체크 유틸의 각 분기
- 실제 시스템 감지 호출은 smoke 테스트 수준

### A2. `get_config()` / `update_config(patch)` 커맨드 (백엔드)

**시그니처 스케치**
- `pub fn get_config(ConfigState) -> AppConfig` (기존 ConfigState에서 clone 반환)
- `pub fn update_config(ConfigState, AppHandle, patch: AppConfigPatch) -> AppConfig`
- `AppConfigPatch`는 모든 편집 가능 필드의 `Option<T>`
  - `editor_command`, `exclude_dirs`, `recent_notes_limit`, `global_shortcut`, `quick_note_folder`
  - `vault_path`는 타입 레벨 제외 (온보딩 전용)
- Some인 필드만 덮어쓰기 (merge) → `save_config` → 반환

**TDD 테스트 대상**
- 부분 patch 머지 로직
- vault_path는 patch에 포함 불가 (컴파일타임 보장)
- 저장 후 roundtrip

### A3. `/settings` 페이지 (프론트)

- 현재 설정 로드 (`getConfig()`)
- 감지된 에디터 드롭다운 + 커스텀 경로 입력
- `exclude_dirs` 간단 태그/칩 editor
- 수치/단축키/폴더명 필드
- 저장 버튼 → `updateConfig(patch)` → 성공 토스트
- 레이아웃 사이드바에 ⚙️ 메뉴 추가

**검증**
- 수동: `/settings`에서 에디터 변경 → "외부 에디터로 열기"가 변경된 에디터로 실행

## Slice B — Claude 도구 뷰

### B1. SKILL.md + 훅 설정 파서 (백엔드)

**시그니처 스케치**
```rust
pub struct ClaudeSkill { name, description, path }
pub struct ClaudeHook  { event, matcher, command, script_path }
pub struct ClaudeTools { claude_md, skills, hooks }

pub fn get_claude_tools(ConfigState) -> Result<ClaudeTools, AppError>
```

**TDD 테스트 대상**
- 스킬 디렉토리에 정상 SKILL.md 2개 → `name/description` 파싱된 2개 반환
- SKILL.md frontmatter 없으면 스킵 (에러 아님)
- settings.json의 `hooks.{Event}[].hooks[].command`에서 `bash .claude/hooks/X.sh` → `script_path` 추출
- `.claude/` 없으면 빈 ClaudeTools 반환 (에러 아님)
- settings.json 파싱 실패 → `hooks: []` + 에러 로그 (사용자 경험 보호)
- CLAUDE.md 존재 여부 반영

### B2. `/claude` 페이지 (프론트)

- `getClaudeTools()` → 3개 섹션 렌더링:
  1. CLAUDE.md: `getNote("CLAUDE.md")` 재사용 + `RenderedNote` 표시
  2. 스킬 카드 그리드 (name + description, 클릭 시 `/view?path=.claude/skills/.../SKILL.md`)
  3. 훅 테이블 (event별 그룹, matcher/command/script_path 컬럼)
- 레이아웃 사이드바에 🤖 Claude 메뉴 추가

**검증**
- 스캐폴드된 볼트: 스킬 14개, 훅 11개 표시
- 스킬/훅 없는 볼트: 안내 문구 표시

## Slice D — 멀티 볼트 관리

### 배경
현재 앱은 단일 `vault_path`만 기억한다. 사용자는 볼트를 바꾸려면 Settings에서 매번 디렉토리 재선택 필요.
대시보드에서 등록된 볼트 목록을 보고 **클릭으로 전환**할 수 있게 확장.

### D1. Config 스키마 확장 + 등록/제거 (백엔드)

**시그니처 스케치**
```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct VaultEntry {
    pub path: PathBuf,
    pub name: String,  // 폴더명 기본, 사용자가 별칭 지정 가능
}

// AppConfig에 추가
pub vaults: Vec<VaultEntry>,   // serde(default)로 migration-safe

pub fn list_vaults(ConfigState) -> Result<Vec<VaultEntry>, AppError>
pub fn switch_vault(ConfigState, VaultState, SearchState, WatcherState, AppHandle, path: String)
    -> Result<VaultStatus, AppError>  // activate_vault 재사용
pub fn remove_vault(ConfigState, AppHandle, path: String) -> Result<Vec<VaultEntry>, AppError>
```

**동작**
- `create_vault`/`connect_vault`가 성공 시 `vaults`에 push (중복 제거, path 정규화)
- `switch_vault`는 `vaults`에 있는 경로만 허용 (존재 확인)
- `remove_vault`는 목록에서만 제거 — 실제 파일/디렉토리는 건드리지 않음
- 현재 활성 볼트를 remove하면 — v0.5 비범위, 에러 반환 (사용자가 먼저 전환해야 함)

**TDD 테스트 대상**
- vaults 필드는 serde_default → 기존 config.json 로드 시 빈 Vec
- create/connect 후 vaults에 추가, 중복 호출 시 1개만 유지
- 경로 정규화 (trailing slash 등)
- switch_vault가 목록에 없는 경로 → 에러
- remove 후 다시 create/connect 가능

### D2. 대시보드 볼트 목록 UI (프론트)

- 대시보드 상단에 "볼트" 섹션
- 각 VaultEntry를 카드로 표시 (이름 + 경로 + 활성 뱃지)
- 클릭 → `switchVault(path)` → 전체 상태 재로드
- 우측에 "+ 추가" 버튼 (Settings로 이동 또는 인라인 flow)
- 활성 볼트 제외한 다른 카드엔 "제거" 작은 버튼

**검증**
- 수동: 2개 볼트 등록 → 대시보드에서 왔다갔다 클릭 → 통계/노트가 올바르게 전환되는지 확인

## 열린 결정 (Spec 작성 전 확정)

- **Obsidian URL 포맷**: `obsidian://open?path=<absolute>` vs `obsidian://advanced-uri?vault=<name>&filepath=<rel>` 중 선택
  - 단순성 위해 `path=` 시작, vault 이름 세팅은 추후
- **스킬 카드 클릭 액션**: 내부 뷰어 우선 (read-only 철학 유지)
- **빈 상태 UX**: `/claude`에서 아무 것도 없을 때 표시할 문구
  - "이 볼트엔 Claude 도구 설정이 없습니다. `.claude/` 디렉토리를 생성하거나 스캐폴드된 새 볼트를 사용하세요."
- **훅 매처 표현**: `Write|Edit` 같은 문자열 그대로 노출 (v0.5 한정)
