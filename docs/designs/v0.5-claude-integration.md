# v0.5 설계 — 에디터 유연성 + 볼트 Claude 도구 뷰

> 상태: 합의 완료, 구현 대기
> 철학: **이 앱은 볼트의 대시보드/뷰어다. 오케스트레이터가 아니다.**
> Claude Code 실행은 사용자가 터미널에서 볼트 디렉토리 안에서 직접 수행.
> 앱은 "이 볼트에 어떤 Claude 도구가 있는지"를 보여주기만 한다.

## 범위

### 축 1 — Settings 페이지 + 에디터 자동 감지

**문제**: `editor_command` 기본값이 macOS VS Code 절대경로로 고정. 사용자 에디터 선택권 없음.

**해결**:
- `/settings` 라우트 추가 — 앱 전역 설정 편집 (볼트별 아님)
- `detect_editors()` Rust 커맨드 — 설치된 에디터 감지 후보:
  - macOS: `/Applications/*.app` 존재 확인 + Homebrew 설치 커맨드 경로 (`which code` 등)
  - Unix: `which code`, `which cursor`, `which zed`, `which nvim`, `which subl`
  - Windows: 표준 설치 경로 스캔
- UI에서 감지 목록 + "사용자 지정 경로" 입력 가능
- Obsidian은 URL 스킴 (`obsidian://open?path=...`)이므로 프리셋 한 칸 추가
- 저장 시 `save_config` → 즉시 반영 (ConfigState write)

**편집 가능 항목 (v0.5)**:
- `editor_command` (에디터 프리셋 또는 커스텀 경로)
- `exclude_dirs` (인덱싱 제외 폴더)
- `recent_notes_limit`
- `global_shortcut` (퀵 노트 단축키)
- `quick_note_folder`

### 축 2 — 볼트 Claude 도구 전용 뷰

**문제**: 스캐폴드는 `.claude/settings.json`, hooks, skills를 생성하지만 앱은 이 존재를 표시하지 않음. 사용자가 "이 볼트에 어떤 Claude 기능이 설치돼있는지" 모름.

**해결**: `/claude` 전용 페이지 — 연결된 볼트의 Claude 도구를 read-only로 표시.

**컨텐츠**:

1. **CLAUDE.md 렌더링**
   - 기존 `renderer::render_note`로 렌더링
   - 페이지 상단 섹션

2. **스킬 목록** (`.claude/skills/*/SKILL.md`)
   - 각 SKILL.md의 frontmatter에서 `name`, `description` 파싱
   - 카드 리스트 (이름 + 설명)
   - 클릭 시 원본 SKILL.md 뷰어로 이동 (편집은 외부 에디터)

3. **훅 매핑** (`.claude/settings.json`)
   - 이벤트(SessionStart / PreToolUse / PostToolUse / Stop / UserPromptSubmit)별 등록된 훅 표시
   - 매처 조건과 명령어 노출
   - 훅 파일 클릭 시 스크립트 보기 (뷰어)

**대시보드(/) 요약 카드 추가**:
- 기존 통계 카드 옆에 "Claude 도구" 카드
- `스킬 N개 · 훅 M개` 한 줄 + `/claude` 링크

## 비범위 (v0.5 에서 안 함)

- ❌ 스킬/훅 실행 (오케스트레이션)
- ❌ 훅 출력 로깅 및 Alerts 대시보드
- ❌ 세션 활동 로그 뷰어
- ❌ 앱 내 편집 (CLAUDE.md, SKILL.md, 훅 스크립트 편집은 외부 에디터에서)
- ❌ 볼트별 설정 (에디터 등은 전역 유지)

## 구현 태스크

1. **Settings 페이지 백엔드**
   - `update_config(partial)` 커맨드 — ConfigState write + save_config
   - `detect_editors()` 커맨드 — 플랫폼별 탐지 로직 + 테스트

2. **Settings 페이지 프론트**
   - `/settings` 라우트
   - 편집 가능 필드 폼
   - 에디터 드롭다운 (자동 감지 + 커스텀)

3. **Claude 도구 뷰 백엔드**
   - `get_claude_tools()` 커맨드 — 반환:
     - `skills: [{ name, description, path }]` (SKILL.md frontmatter 파싱)
     - `hooks: [{ event, matcher, command, script_path }]` (settings.json 파싱)
     - `claude_md: string | null` (경로만, 렌더링은 기존 get_note 재사용)
   - 파싱 실패 시 Result 에러 (필드가 없거나 JSON 깨졌을 때)

4. **Claude 도구 뷰 프론트**
   - `/claude` 라우트
   - CLAUDE.md 섹션 + 스킬 카드 + 훅 테이블
   - 대시보드 요약 카드

## 열린 질문

- 스킬 SKILL.md의 `name`/`description` 파싱 기준은? (claude-code 공식 포맷 따를 것)
- 에디터 자동 감지는 매 앱 실행마다 vs 사용자가 "재검색" 버튼 누를 때? → 후자가 가벼움
- `/claude` 페이지 접근은 레이아웃 사이드바에 메뉴 추가로 노출
