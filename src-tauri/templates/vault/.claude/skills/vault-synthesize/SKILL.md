# 지식 종합

특정 주제에 대한 흩어진 노트들을 모아 하나의 종합 정리 노트를 생성한다.
"/vault-synthesize", "종합 정리", "노트 통합" 등의 키워드에 반응.

## 절차

1. **주제 확인**: 사용자에게 종합할 주제(태그, 키워드, 폴더)를 확인
2. **소스 노트 수집**:
   - 해당 주제 관련 노트를 태그, 키워드, 링크 기반으로 검색
   - 후보 노트 목록을 사용자에게 보여주고 포함/제외 확인
3. **내용 분석**:
   - 각 소스 노트의 핵심 내용 추출
   - 중복/유사 내용 병합
   - 시간순 또는 논리 구조로 재배열
4. **종합 노트 생성**:
   - 타입: `til` (dev/ 폴더) — 주제가 기술적일 경우
   - status: `evergreen`
   - 소스 노트들을 `## 관련 노트`에 전부 링크
   - 각 소스에서 핵심을 추출하여 체계적으로 정리
5. **소스 노트 업데이트**: 각 소스 노트의 `## 관련 노트`에 종합 노트 역링크 추가
6. **MOC 등록**: 관련 MOC에 종합 노트 추가

## 종합 노트 구조

```markdown
---
type: til
created: YYYY-MM-DD
tags:
  - <주제 태그>
status: evergreen
sources:
  - <소스 노트 목록>
---

# <주제명> 종합 정리

## 개요
(주제에 대한 1-2문단 요약)

## 핵심 내용
(소스 노트에서 추출한 내용을 논리적으로 재구성)

## 관련 노트
- [[source-note-1]]
- [[source-note-2]]
- ...
```

## 규칙

- 소스 노트의 원문을 삭제하지 않음 (종합은 상위 레벨 요약)
- 소스가 3개 미만이면 종합의 가치가 낮으므로 사용자에게 안내
- 이미 해당 주제의 종합 노트가 있으면 새로 만들지 않고 기존 노트 업데이트 제안
