# 웹 클리핑

URL을 받아 웹 페이지 내용을 요약하고 clipping 노트로 저장한다.
"/vault-clip", "클리핑", "웹 저장", "URL 메모" 등의 키워드에 반응.

## 절차

1. **URL 확인**: 사용자에게 URL을 받음 (여러 개 가능)
2. **페이지 수집**: WebFetch로 페이지 내용 가져오기
3. **메타데이터 추출**:
   - 제목, 저자, 발행일
   - 주요 키워드/태그 후보
4. **내용 요약**:
   - 핵심 내용 3-5문장 요약
   - 주요 포인트 불릿 리스트
   - 인용할 만한 문장이 있으면 블록인용으로 포함
5. **노트 생성**:
   - `_templates/tpl-clipping.md` 기반
   - 파일명: `YYYY-MM-DD-<제목-kebab-case>.md`
   - 폴더: `inbox/`
   - frontmatter의 source에 URL, author에 저자, clipped에 오늘 날짜
6. **태그 제안**: 기존 볼트 태그 목록과 대조하여 적합한 태그 제안, 사용자 확인
7. **관련 노트 연결**: vault-new와 동일한 로직으로 관련 노트 검색 + 링크

## 규칙

- 원문을 그대로 복사하지 않음 (요약 + 핵심 포인트)
- 사용자가 추가 메모를 남기고 싶으면 `## 내 생각` 섹션에 기록
- 한 번에 여러 URL 처리 가능 (순차 생성)
- WebFetch 실패 시 URL만 기록한 빈 클리핑 생성 + 사용자에게 안내
